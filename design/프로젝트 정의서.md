# 프로젝트 정의서
<br>

## 개요
### 프로젝트 

데브캠프 수강생 및 관리자용 행정 인트라넷 시스템

#### 목표

- 수강생들에게 외출/조퇴, 휴가, 공가,문서 발급 신청 등 간편하게 행정 관련 신청할 수 있는 기능 제공
- 관리자들에게 수강생 관리, 신청 승인, 문서 발급 등의 기능 제공
- <u>**사용자 친화적인 웹 인터페이스 제공**</u>

#### 사용자 정의
- 수강생
- 관리자

## 요구사항

### 수강생용 인트라넷

1. 외출/조퇴 신청

   - 신청서 작성(사유만 작성) 후 수강생 본인이 입실/퇴실 버튼으로 기록 관리

 2. 휴가 신청

    - 2주 전 휴가 날짜 작성 후 출석대장, 휴가계획서 제출

 3. 공가 신청

    - 공가 신청서(날짜, 사유) 제출
    - 공가 종료 후 출석대장, 증빙서류 제출

 4. 문서 발급

    - 국취서류(참여확인서, 수강증명서, 출석부) 등 요청

 5. 문의

    - 행정 문의 작성 및 제출

6. 기업 공지 모음

    - 구인 광고 열람 가능

 7. 신청 현황

    - 본인의 행정 신청 현황 열람 가능

<br>

### 관리자용 인트라넷

1. 수강생 리스트
    - 수강생 관리

2. 외출/조퇴 신청 관리
    - 수강생의 외출/조퇴 신청 확인 및 승인
    - 입/퇴실 체크 확인 및 출결 기록 반영

3. 휴가 신청 관리
    - 수강생의 휴가 신청 확인
    - 휴가 날짜, 출석대장 및 휴가계획서 검토 및 승인
    - 출결 기록 반영

4. 공가 신청 관리
    - 수강생의 공가 신청 확인 및 임시 승인
    - 공가 종료 후 출석대장, 증빙서류 검토 및 최종 승인
    - 출결 기록 수정

5. 문서 발급 요청 관리
    - 수강생의 문서 발급 요청 확인 및 처리
    - 문서 발급
    - 발급 완료 시 문서 전달

6. 문의 관리

   - 수강생의 행정 문의 확인 및 답변 작성

7. 기업 공지 모음

   - 구인 광고 업로드

8. 신청 현황

   - 수강생들의 행정 관련 신청 사항들에 대한 현황을 알려줌

<br>

## 시스템 아키텍처

### 아키텍처 개요

- **프론트엔드:** Html. Css, JavaScript
- **백엔드:** Node.js (Express.js)
- **데이터베이스:** MySQL

### 모듈 구성

- **프론트엔드:** 사용자 인터페이스, 데이터 시각화, API 연동
- **백엔드:** RESTful API, 비즈니스 로직, 데이터베이스 연동
- **데이터베이스:** 수강생 정보, 신청 정보, 출결 정보


<br>

## 데이터베이스 설계

- **테이블:** 수강생, 외출/조퇴 신청, 휴가 신청, 공가 신청, 문서 발급 요청, 기타 문의, 기업 공지, 신청 현황
- **컬럼:** 각 테이블의 주요 컬럼 정의


<br>

## 인터페이스 설계

- **프론트엔드-백앤드-데이터베이스:**  Node.js를 사용한 데이터 연동

<br>

## 설계 상세

### 사용자 스토리
<br>

#### 수강생용

1. 외출/조퇴 신청
    - 수강생으로서, 외출/조퇴 신청한 후 입실/퇴실 기록을 관리하고 싶습니다.

2. 휴가 신청
    - 수강생으로서, 2주 전에 휴가를 신청하고 출석대장과 휴가계획서를 제출하고 싶습니다.

3. 공가 신청
    - 수강생으로서, 공가 신청서를 제출하고 공가 종료 후 출석대장과 증빙서류를 제출하고 싶습니다.

4. 문서 발급
    - 수강생으로서, 국취서류 등을 요청하고 싶습니다.

5. 문의
    - 수강생으로서, 행정 관련 문의를 작성하여 제출하고 싶습니다.

6. 기업 공지 모음
    - 수강생으로서, 구인 광고를 열람하고 싶습니다.

7. 신청 현황
    - 수강생으로서, 행정 신청 현황에 대해 알고 싶습니다.


#### 관리자용

1. 수강생 리스트
    - 관리자로서, 수강생을 관리하고 싶습니다.

2. 외출/조퇴 신청 관리
    - 관리자로서, 수강생의 외출/조퇴 신청을 확인하고 승인하고 싶습니다.
    - 관리자로서, 입/퇴실 체크 내역을 확인하고 출결 기록에 반영하고 싶습니다.

3. 휴가 신청 관리
    - 관리자로서, 수강생의 휴가 신청을 확인하고 승인하고 싶습니다.
    - 관리자로서, 휴가 날짜, 출석대장, 휴가계획서를 검토하고 승인하고 싶습니다.
    - 관리자로서, 휴가 신청이 출결 기록에 반영되게 하고 싶습니다.

4. 공가 신청 관리
    - 관리자로서, 수강생의 공가 신청을 확인하고 임시 승인하고 싶습니다.
    - 관리자로서, 공가 종료 후 출석대장, 증빙서류를 검토하고 최종 승인하고 싶습니다.
    - 관리자로서, 공가 신청이 출결 기록에 반영되게 하고 싶습니다.

5. 문서 발급 요청 관리
    - 관리자로서, 수강생의 문서 발급 요청을 확인하고 처리하고 싶습니다.
    - 관리자로서, 문서 발급 완료 후 수강생에게 문서를 전달하고 싶습니다.

6. 문의 관리
    - 관리자로서, 수강생의 행정 관련 문의를 확인하고 답변하고 싶습니다.

7. **기업 공지 모음**
    - 관리자로서, 구인 광고를 작성하여 업로드하고 싶습니다.

8. 신청 현황
    - 수강생들의 행정 관련 신청 사항들에 대한 현황을 알려주고 싶습니다.
